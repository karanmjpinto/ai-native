A field guide

Adding AI changes nothing until you redesign the work.

The first electric factories bolted a motor onto a steam-era floor and saw no gain at all. The payoff came decades later, when owners tore up the layout and built around the motor. AI is at that same moment now. This is a short guide to the rebuild.

Read the guide ↓
1.0 Why

A faster version of yesterday is not the goal.

Bolt AI onto the way you work today and you get exactly that: yesterday, run a little quicker. The org chart, the hand-offs, the meetings — all of it was built for people doing the work by hand. Speeding up a process that shouldn't exist is not progress.

The gain shows up when you ask a harder question. Not "where can AI save us time?" but "if machines can do the doing, what should the work even be?" That is the rebuild. Everything below is how to start it.

2.0 Moat

Your moat is a loop, not a tool.

A rival can buy the same model and copy your best feature by Friday. What they can't copy is how fast you learn. That speed comes from a loop with three parts, each feeding the next.

  • System. Build something that reliably ships work, not slides about work.
  • Speed. Ship it faster than anyone else can — small, often, in the open.
  • Signal. Read what shipping taught you, then feed it back to the system.

Each turn of the loop makes the next turn quicker. Run it long enough and the gap stops being a gap. It becomes a moat.

3.0 Scarce

When intelligence is free, four things stay scarce.

Agents make execution cheap, and before long everyone will have them — the same models, shipping the same features. When the doing costs almost nothing, doing it is no longer where the advantage lives. So the question that outlasts the tools is this: when intelligence is free, what is still hard to get?

  • Context. What your organisation has learned that no one else can see. Earned from your own work — a rival can't buy it.
  • Attention. One place people actually come to do the work. Hard to win, harder to hold.
  • Trust. Being measured on the outcome, not the pitch. Skin in the game, not slideware.
  • Brand. A reputation that compounds, so each win makes the next one easier.

None of these are the agents. The agents are table stakes now. The moat is everything they can't hand you — and context, the first of the four, is the one you can build on purpose. The rest of this guide is how.

4.0 People

Everyone becomes a manager.

When agents handle the execution, people move up a level. You stop doing the task yourself and start directing it — the way a good manager directs a team. So your hours should move with you: most of them onto strategy and judgement, far fewer on execution. The agents take that part.

  • YouStrategy. Decide what's worth doing, and what isn't. Give this more of your time, not less.
  • AgentsExecution. Hand it off with enough context to act without you. This is the part that should shrink.
  • YouJudgement. Check the work, catch what's wrong, own the call.
5.0 Agents

An agent is only as good as four things.

Autonomy isn't magic. An agent you can actually trust to run on its own stands on four legs. Get all four right and you can step back. Miss one and you're back to doing the work yourself.

  • Goals. A clear outcome it's working toward.
  • Skills. The know-how to pursue that outcome.
  • Tools. The means to actually take action.
  • Context. Enough of the situation to act well.
6.0 Context

Context is the engine, and it runs in a loop.

Of the four legs, context is the one that compounds. It isn't a file you write once. It's a cycle the whole organisation runs, and each pass leaves the next agent a little smarter.

  1. Capture what happens as work is done.
  2. Curate the small part of it that matters.
  3. Store it where any agent can reach it.
  4. Execute the next task on that better footing.
  5. Experience the result — which becomes the next capture.

Traces from the work and signals from the market both feed back into capture. The loop tightens every cycle. That tightening is the whole game.

For yourself The same loop, on top of the apps you already use

You don't need new software to start. If you already keep notes in Obsidian, plain markdown, or Apple Notes, you have capture and store covered. The work is laying a thin layer over them, so an agent can read what you know — and write back to it.

  1. Capture in plain markdown — one folder, so anything can read it.
  2. Curate & store by pointing a memory tool like Mem0 or SuperMemory at that folder.
  3. Execute by giving your agent read access to the index it builds.
  4. Experience by saving what you learn back as notes — the loop closes, at the size of one person.
